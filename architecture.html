Request Data Flow
1
User changes a filter in FilterBar
Four global filters: Time Period, Location (Warehouse), Category, Order Status. State lives in useFilters() composable as Vue refs — shared across all views as a singleton.
2
View component watch() fires
Each view watches the relevant filter refs. On change, it calls its loadData() function, sets loading.value = true, and clears any previous error.
3
api.js builds URL params and sends HTTP request
Filters mapped to query params. Values set to "all" are omitted entirely. Axios sends GET request to the FastAPI backend.
GET http://localhost:8001/api/orders?warehouse=San+Francisco&month=2025-01&status=Processing
4
FastAPI applies filters to in-memory data
apply_filters() sequentially filters the in-memory list by warehouse → category → status. filter_by_month() then matches YYYY-MM dates or expands quarters (Q1-2025 → Jan/Feb/Mar).
5
Pydantic validates and serializes the response
FastAPI uses Pydantic v2 models as response_model. Fields are validated and unknown fields are stripped before serialization to JSON.
6
Vue stores response in ref, computed properties react
Raw API response stored in a ref (e.g. allOrders). Any local filtering (e.g. search box) lives in a computed() derived from that ref. Template re-renders automatically.

Key Design Patterns
Global Filter State (Singleton Composable)
useFilters() stores filter refs outside the component tree. All views share the same reactive state — changing a filter in FilterBar instantly propagates to every view's watcher.
Raw Ref + Derived Computed
API responses are stored as raw refs (allOrders, inventoryItems). Search, sort, or local filtering are expressed as computed() properties — never mutating the source ref.
Backend Sequential Filtering
Filters are applied in sequence: warehouse → category → status → month. Each step skips if the value is None or "all". Inventory does not support the month filter (no time dimension).
In-Memory Mock Data (No Database)
mock_data.py loads all 7 JSON files at startup into module-level Python lists. Every request filters these in-memory lists. There is no persistence layer — restarting the server resets all state.
Pydantic Response Validation
Every endpoint uses response_model. Pydantic v2 validates and strips unexpected fields before serialization, keeping the API contract stable even as JSON data evolves.
i18n with Locale-Aware Currency
useI18n() handles EN/JA translation with localStorage persistence. Currency automatically switches (USD ↔ JPY) based on locale, and all product names, warehouses, and user fields are translated.
